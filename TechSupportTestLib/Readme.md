Алгоритм знаходженя мінімальної кількості зустрічей:

1. Ділимо масив на 3 змінні: змінну з кількістю їжаків бажаного кольору та 2 змінні з кількістю їжаків інших кольорів. Назвемо їх к1 і к2.
2. Заносимо у змінну "кількість зустрічей" найбільшу кількість їжаків не головного кольору.
3. Він більшої кількості їжаків не головного кольору віднімаємо меншу, та додаємо в двічі більшу кількість їжаків до головного кольору.
4. Якщо кількісь їжаків другорядного кольору, яка залишилася, не ділиться націло на 3, то рішення не буде. 
	1. Пояснення: нехай у нас залишилося 3x їжаків одного кольору після усіх зустрічей. Тоді ми можемо об'єднати x з головним кольором і отримати 2x їжаків першого кольору і 2x їжаків другого кольору, яких ми зможемо об'єднати. Але якщо x не ділиться націло на 3, то об'єднання не вийде.
5. Якщо кількість їжаків другого кольору після цього не 0 і кількість їжаків головного кольору 0 (це може бути якщо нам дали 0 їжаків головного кольору, 0 першого і x другого) то рішення не буде.
6. Якщо ми дійшли сюди, то рішення є. Кількість зустрічей буде дорівнювати змінній "кількість зустрічей", або ж найбільшому вхідному значенню кількості їжаків другорядного кольору.
